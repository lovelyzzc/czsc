# Surge Regime 回踩买点研究（2026-06-10）

本文接续 `SURGE_REGIME_MARKET_STATE_WALKFORWARD_2026-06-10.md`。

## 研究目标

上一轮验证表明：`anticipate` 模式叠加市场状态门
`high20_ratio > 0.12 & ew_index_above_ma20` 后，2024+ OOS 聚合毛超额通过 t>=2，
但净年化只有 +1.6%，2026YTD 净收益仍弱。

审计报告中的第二条线索是：`anticipate` 信号延迟 5-7 天入场表现优于立即入场。
本轮目标就是验证：

> 在已经通过研究层验证的市场状态门内，离开中枢后第 5 / 7 天仍处于上行家族时，
> 是否构成更好的回踩买点。

## 方法

不修改 FSM，不修改 `surge_onset`，直接使用
`scripts/_output/surge_candidates/candidates.parquet` 中已有的多 delay 候选。

固定条件：

- 模式：`anticipate`；
- 信号门：沿用原始 `surge_onset` 门控；
- 硬过滤：成交额 >= 1 亿、止损带 8-20%、入场开盘不贴近涨停、历史 ST / 退市剔除；
- 市场状态门：`high20_ratio > 0.12 & ew_index_above_ma20`；
- 组合：10 槽实盘镜像；
- 成本：买 0.15% / 卖 0.25%；
- beta 对照：同日、同成交额十分位随机对照，且同一笔交易稳定匹配同一组对照。

新增脚本：

- `scripts/surge_pullback_entry_research.py`

运行命令：

```bash
uv run --no-sync python scripts/surge_pullback_entry_research.py
```

输出：

- `scripts/_output/surge_pullback_entry/report.md`
- `scripts/_output/surge_pullback_entry/summary.json`

- 2026YTD 样本只有 44 笔，年化数值不可过度解读，应优先看单笔净均值与毛超额。

## 前推选择验证

选择规则：

1. 每个测试年只使用更早年份；
2. 在固定变体 `state_delay0 / state_delay5 / state_delay7 / state_delay5_first_else7`
   中选择历史毛超额 t 值最高且均值为正的变体；
3. 历史样本少于 60 笔的变体不参与选择；
4. 不使用测试年信息调参。

## 本轮判定

`anticipate + 市场状态门 + delay5 回踩买点` 是目前最有证据支持的变体。

它满足：

- 2024+ OOS 毛超额为正且 t>=2；
- 单笔净均值从立即入场的 +0.57% 提升到 +6.19%；
- 前推选择能在 2025 前选出 `delay5`，且 2026YTD 相对立即入场仍有改善；
- 最大回撤没有恶化。

但仍需保留约束：

- 不要把 `delay7` 纳入默认逻辑；
- 不要相信稀疏组合年化，尤其是 2026YTD；
- 当前样本仍偏少，实盘前需要在 daily_scan 镜像中做“真实候选等待 5 天”的状态管理验证，
  而不是只读离线候选表。

## 下一步建议

下一轮可以进入工程镜像验证：

1. 在 daily_scan skill 中增加实验模式，不改变默认行为；
2. 实验模式只输出 `anticipate` 信号后第 5 个交易日仍满足：
   市场状态门、上行家族、硬过滤的候选；
3. 用最近若干日真实扫描结果与离线 `candidates.parquet` 交叉核对；
4. 通过后再考虑把它作为“实验标签”而不是默认买入建议。
