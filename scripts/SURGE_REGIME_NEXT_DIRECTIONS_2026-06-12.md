# Surge Regime 后续迭代方向备忘（2026-06-12，五轮研究收官后）

背景：入场阈值 / 选股因子 / 退出规则 / 组合容量四层已全部冻结或按预声明定格
（见 `SURGE_REGIME_SELECTION_AUDIT_2026-06-11.md`、`SURGE_REGIME_FACTOR_RESEARCH_2026-06-11.md`、
`SURGE_REGIME_EXIT_RESEARCH_2026-06-11.md`、`SURGE_REGIME_CAPACITY_RESEARCH_2026-06-12.md`）。
**「日线量价 × 选股阈值」空间内无剩余优化维度**——本文档记录的是该边界之外的候选方向，
按性价比排序，供后续迭代参考。任何新变体动手前必须走完整流程：
candidates.parquet 敏感性 → 工程镜像核对 → 预声明判定（标准先于运行写定，跑完不改）。

策略当前定性：均值靠尾部、中位数为负的彩票组合；全史诚实年化 +2.8%/年、MDD 43.9%
（全日历口径，对外引用只用这个数）；定位 = 观察/出清工具 + delay5 前向数据积累。

## 一、已预声明的重检点（零新开发，等触发条件，优先级最高）

这两项是研究中明确留了"重开钥匙"的，不算新优化，触发即执行：

### 1.1 bd_confirm2（破坏二次确认）重检

- **现状**：唯一 pair 级全面改善的退出变体（超额中位数 -0.74 → -0.22）；第五轮
  @20/30/40 槽 OOS 中位数五轮来首次转正（+0.74 / +0.88 / +0.62，t≥4.25），
  但被 IS 条款拦下（IS 中位数 -2.49 < full 的 -1.93，改善仅存在于 2024+ 环境）。
- **唯一重开条件**：delay5 前向真实样本（次日开盘成交模拟口径，见 delay5_scan
  前向进度读数）**累计 ≥60 笔**后，按第五轮同式重检。
- **判定式不变**：OOS 中位数>0 且 t≥2 且 IS 中位数不劣于 full 参照。

### 1.2 市场门收紧 high20_ratio 0.12 → 0.15/0.20

- **现状**：选股条件审计中市场门是唯一有分离力的条件；收紧到 0.15/0.20 后
  交易级中位数转正（+0.34 / +0.78）但均值与样本量下降，未过预声明关。
- **触发条件**：前向样本若呈同向（收紧人群中位数更优），这是**第一个重检点**。
- 注意：与 1.1 共用前向日志，两者都依赖 `picks_exp_delay5_*.parquet` +
  `market_state_live.parquet` 的持续积累，门关空集属常态、不重置计数。

## 二、现有数据内的真正缺口：涨停密集型主升的互补探测器

因子研究副产品指出的结构性盲区，是**唯一既不依赖新数据、又不与冻结决议冲突、
还直击收益来源**的方向：

- **事实**：FSM 状态机漏掉涨停密集型最强主升——000636 整史 0 个 surge 事件，
  但 2025 年实走 +63.8%/40 根；FSM 在 5→10→5 间振荡进不了主升态。
  策略收益全靠尾部，而现行探测器结构性捕获不到最强的那类尾部。
- **做法**：
  1. ground truth 改用价格口径标签（未来 40 根最大涨幅 ≥30%，基率 12.8%），
     不用 `find_surges` FSM 事件标签；
  2. 针对涨停密集路径设计互补探测器（候选特征：连板计数、一字/T字板占比、
     缩量板、板后缩量回踩），与现有 anticipate **并联**而非替换；
  3. 全流程过 dump 敏感性 + 镜像 + 预声明（注意涨停票的次日开盘可成交性——
     gap ≥ 板限-0.3% 放弃规则会天然砍掉一部分样本，预声明里要先想清楚口径）。
- **风险提示**：该人群与"热度因子 IC 符号反转"悖论（越热越可能主升、典型超额越差）
  正面相撞，预声明必须以中位数/分布形态为主判定，不准只看均值。

## 三、新数据源（因子空间重开的前提，按契合度排序）

记忆与因子研究结论：日线量价 17 因子已测完，重开因子层必须引入新数据源。

1. **板块/行业联动**（成本最低，日线 + 行业分类即可）：个股层面热度因子 IC 反转
   已证，但**板块层面的条件信息未测**——"谁的尾部会兑现"可能取决于所在题材是否
   处于扩散期。候选：行业内信号密度、板块涨停家数变化率、行业等权指数相对强度。
2. **分钟级（只做执行层，不碰选股）**：次日开盘入场 → 分钟级择时（竞价行为、
   前 30 分钟确认）。对中位数 -0.19% 的策略，执行层省 0.2-0.3% 即质变。
   不重开选股因子，避免维度爆炸。
3. **龙虎榜/资金流**：游资席位与涨停密集型主升强相关，与方向二互补。
4. **基本面**：价值最低（5-40 天持有期的量价策略），仅作排除项记录。

## 四、换问题而不是换参数：状态机的卖出端价值

未被五轮任何一轮覆盖的角度：**把状态机反过来用**——破坏/背驰态对已有持仓的
风险预警（避损端）价值从未量化。

- 线索：退出分布中 state（背驰/破坏）退出占 ~75%，远超 trail18（~13%）和
  sl2（~6%）——状态机识别"该走了"的能力可能强于识别"该进了"。
- 研究问题：对任意持仓（不限于本策略入场的票），状态机发出破坏/背驰后
  N 日的条件收益分布 vs 无条件分布；若分离显著，可独立成"持仓风险预警"工具，
  与"观察/出清工具"的现有定位一致。
- 买入信号 alpha 与卖出信号 alpha 是两个独立问题，前者已证伪不影响后者。

## 五、明确不再碰的（已有审计结论，重开即数据挖掘）

- priority 排序权重（delay0 反向、delay5 正向但非优化对象）、freshness（0-10 天不衰减）
- 止损带 8-20%（带外人群 +2.58 仍正，没在扔金子）、trail 18%（非刀刃）
- delay 取值（2-5 皆优无刀刃）、above_zg 门（状态 5 定义内生，完全冗余）
- time-stop（反向：尾部赢家 23.2% 第 5 天仍低于入场价，砍死交易=砍彩票）
- 槽数（预声明规则已定格 30）、门控 ±20%（不敏感）
- walk-forward LR / IC 加权组合选股（OOS AUC≈0.5；冷门倾斜避开彩票=避开收益来源）

## 六、流程提醒（每个新方向通用）

1. 预声明判定标准先于运行写进脚本 docstring，跑完不改；
2. 彩票型分布下以**中位数 + 分布形态**为主判定，均值不可外推；
3. OOS 强于 IS 先怀疑 beta（2024Q4 起测试窗含 A 股大涨 + 微盘 beta）；
4. 稀疏组合不用 WeightBacktest 年化，读 pair 级统计或全日历诚实年化；
5. 落地前过工程镜像核对（`surge_delay5_mirror_check.py` 模式），
   live 路径与研究 dump 必须逐字节一致。
